Technical Interview Exercises (Beta)

Use Technical Interview Exercises (TIE) to practice solving problems using real code.
Get conversational feedback as you progress toward a solution.

How to get started

  1. Download the zip file *
  2. Unzip the zip file
  3. In the unzipped folder, open the file client/question/question.html in a web browser
  4. Use the coding window on the right to code a solution and click "Get Feedback" whenever you would like feedback

To stop the TIE application, simply close the browser window. To remove TIE from your computer, delete the downloaded files from steps 1 and 2.

* By downloading and using TIE, you agree to the terms contained in the TIE LICENSE file.

Troubleshooting

"The download link doesn't work."

If the link above does not work, try opening https://github.com/google/tie/releases and under the latest (newest) version, click "Source code (zip)".

"I can't open the question.html file."

Chromebook users: Chromebooks may not be able to unzip files by default. To enable this on Chromebooks:

  1. In your browser, go to chrome://flags
  2. Scroll down to "New ZIP unpacker" and select "Disabled"
  3. Use the "RESTART NOW" button to restart the browser or go to chrome://restart
  4. Try the "How to get started" instructions again

Non-Chromebook users: Make sure your browser is up-to-date. We have seen cases where browsers stop opening local html files but are able to after updating to the latest version.

Frequently Asked Questions

"Why is there only one question? Will there be more?"

The TIE team is currently focused on getting the user experience right and prioritizing features that make TIE unique, such as the feedback it provides to users. We hope to add more questions in the future.

"Why does everything look so plain? Why doesn't it look like a Google product?"

This version of TIE is open source, and while it is being developed by Google employees, it is not an official Google product. We want to maintain a simple user interface while focusing on features that enhance the interactive user experience.

"Why did you create TIE?"

We think users might find it helpful to have a tool that tries to emulate the interaction that one may experience during a coding interview at places like Google--specifically, trying to come up with code to solve a problem while getting tips and prompts from the interviewer.

In general, we believe hands-on practice is one of the best ways to learn and cement learning. People don't always have someone sitting next to them to help out when they get stuck. TIE can provide users with guidance as they work to solve coding challenges.
